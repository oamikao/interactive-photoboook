My Article HTML

April 28, 2026

After reading through the article on HTML elements, I think that I did a good job focusing on keeping semantic elements for structure and readability, making sure that each block of text or header/subtitle is properly defined when needed. I made sure that relevant content stayed within the section tags of the main body, and additional but not necessary info in the footer tags. I didn’t add any div tags to the actual HTML so all the styling decisions are in the css stylesheet.

The only small detail I had to make a solution for was styling the subtitle for the image, which meant I had to go back into my index and add a class to my paragraph tags so make sure I was only adjusting that one line. When styling, I was aiming for an archival/document look since my topic was about an old medium of art, so I created that effect with color and typography, but still definitely could’ve added more to emphasize that style. I experimented with drop shadows and paragraph/nav padding and margins to get the feel that I wanted, and found that it was actually a bit hard to make the article visually interesting with the limited tools that I had.

Playing around with colors and text/border decorations was fun to do but felt very limited, and looking back, I definitely could have added background images to or icons that were relevant to the look I was going for. Overall though, it was fun to see the article have more personality once styling was added to it.
